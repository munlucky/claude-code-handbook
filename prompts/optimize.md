# Optimize Prompts

성능 최적화 요청 시 사용하는 프롬프트 스니펫입니다.

## Performance Optimization

### 일반 최적화
```
이 코드를 최적화해줘:
- 현재 문제점 분석
- 최적화 방안 제시
- Before/After 비교
- 트레이드오프 설명
```

### 특정 목표 최적화
```
이 코드를 [목표]에 맞게 최적화해줘:
목표: [속도 / 메모리 / 번들 크기 / 가독성]
제약: [현재 구조 유지 / 의존성 최소화 / 호환성]
```

### 쿼리 최적화
```
이 SQL 쿼리를 최적화해줘:
- 실행 계획 분석
- 인덱스 제안
- 쿼리 재작성
- 예상 성능 개선
```

## Code Quality

### 리팩토링
```
이 코드를 리팩토링해줘:
- 가독성 개선
- 중복 제거
- 단일 책임 원칙 적용
- 테스트 용이성 개선
동작은 동일하게 유지해줘.
```

### 모던 문법으로 변환
```
이 코드를 최신 문법으로 변환해줘:

- 불필요한 코드 제거
```

## Bundle Optimization

```
번들 크기를 줄이는 방법 분석해줘:
- 현재 번들 분석
- 큰 의존성 식별
- Tree shaking 개선
- Code splitting 제안
- 대체 라이브러리 제안
```

## Database Optimization

```
이 데이터베이스 스키마/쿼리를 최적화해줘:
- 정규화/비정규화 검토
- 인덱스 전략
- 쿼리 패턴 분석
- 캐싱 전략
```

## API Optimization

```
이 API를 최적화해줘:
- 응답 시간 개선
- 페이로드 크기 최소화
- 캐싱 전략
- Rate limiting
```

## Request Context

```
최적화 요청 시 다음 정보 포함:
- 현재 성능 수치 (응답 시간, 메모리 등)
- 목표 성능
- 제약 조건
- 환경 (Node 버전, 브라우저 등)
```

## Response Format

```
최적화 결과를 다음 형식으로 알려줘:
1. 문제점 분석
2. 해결 방안 (우선순위순)
3. 각 방안의 예상 효과
4. 구현 코드
5. 측정 방법
```
