# Explain Prompts

코드나 개념 설명을 요청할 때 사용하는 프롬프트 스니펫입니다.

## Code Explanation

### 기본 설명
```
이 코드가 무엇을 하는지 설명해줘:
- 전체 흐름
- 각 부분의 역할
- 왜 이렇게 작성했는지
```

### 상세 분석
```
이 코드를 분석해줘:
1. 입력과 출력
2. 시간/공간 복잡도
3. 엣지 케이스 처리
4. 잠재적 문제점
5. 개선 가능한 부분
```

### 초보자용 설명
```
프로그래밍 초보자에게 설명하듯이 이 코드를 설명해줘:
- 전문 용어는 쉬운 말로
- 비유나 예시 사용
- 단계별로 차근차근
```

## Architecture Explanation

### 시스템 설명
```
이 아키텍처를 설명해줘:
1. 각 컴포넌트의 역할
2. 데이터 흐름
3. 왜 이런 구조를 선택했는지
4. 장단점
```

### 디자인 패턴 설명
```
이 코드에서 사용된 디자인 패턴을 설명해줘:
- 어떤 패턴인지
- 왜 여기서 사용했는지
- 다른 패턴과 비교
```

## Concept Explanation

### 기술 개념
```
[개념]을 설명해줘:
1. 정의
2. 왜 필요한지 (어떤 문제를 해결하는지)
3. 어떻게 동작하는지
4. 실제 사용 예시
5. 장단점
```

### 비교 설명
```
[A]와 [B]의 차이점을 설명해줘:
- 각각의 특징
- 언제 어떤 것을 사용하는지
- 실제 코드 예시
```

## Error Explanation

```
이 에러를 설명해줘:
1. 에러 의미
2. 왜 발생했는지
3. 어떻게 해결하는지
4. 어떻게 예방하는지
```

## Response Format Request

```
설명할 때 다음 형식으로 해줘:
- 한 줄 요약
- 상세 설명
- 코드 예시
- 관련 자료 링크 (있다면)
```
