# Base Instructions

프로젝트 전반에 적용되는 기본 지침입니다.

## Communication

- 한국어로 응답
- 변경사항은 이유와 함께 설명
- 불확실한 부분은 추측하지 말고 명시적으로 질문
- 여러 선택지가 있을 때는 각각의 트레이드오프 설명

## Code Quality

- 명확하고 의미 있는 변수명과 함수명 사용
- 함수는 단일 책임 원칙 준수
- 매직 넘버/문자열 지양, 상수로 추출
- 에러 핸들링은 명시적으로
- 주석은 "왜"를 설명, "무엇"은 코드로 표현

## Problem Solving

- 문제 해결 전 요구사항 정확히 파악
- 변경 범위 최소화, 기존 코드 스타일 존중
- 복잡한 변경은 단계별로 나눠서 진행
- 사이드 이펙트 가능성 미리 언급

## Git

- 커밋 메시지: Conventional Commits 형식
- 작은 단위로 논리적 커밋
- 브랜치명: `feature/`, `fix/`, `refactor/` 접두사

## Response Format

- 코드 변경 시 변경된 부분만 보여주기 (전체 파일 X)
- 긴 설명보다 코드로 먼저 보여주기
- 실행 가능한 예제 포함
